## 본질

**Logotekton(로고텍톤)** 은 그리스어 *Λόγος*(Logos, 체계·이성·지식)와 *τέκτων*(Tekton, 짓는 자·건축가)의 결합어입니다.

'Architecture'의 어원이 *Archi-tekton*('으뜸 장인')이듯, Logotekton은 **'지식의 으뜸 장인'** 을 뜻합니다.

> **우리는 건물을 짓지 않습니다. 우리는 건물을 짓는 *방법*을 짓습니다.**
>
> *We do not build buildings. We build the way buildings are built.*

---

## 철학

```
Logos    →  체계로서의 지식       Knowledge as System
Tekton   →  짓는 행위             The Act of Building
Onto     →  존재의 정의           Defining What Is
Techne   →  이론과 현장의 연결    Theory meets Practice
```

우리는 다음을 믿습니다:

1. **건축은 데이터다** — 흩어진 도면·문서·경험은 모두 구조화 가능한 지식이다.
2. **지식은 건축이다** — 온톨로지는 단순한 데이터가 아니라, 사고의 구조물이다.
3. **장인의 경험은 자산이다** — 암묵지를 명시지화하는 것이 진정한 AX이다.
4. **AI는 도구가 아니라 동료다** — 에이전트는 판단을 보조하는 협업자다.

---

## 비전

설계도 위의 선, 시공 현장의 경험, 모듈의 규격, BIM의 데이터 — 흩어진 산업 지식을 **온톨로지로 체계화**하고, **AI 에이전트로 지능화**합니다.

건물을 짓는 일은 오래된 일이지만, *건물을 짓는 방법을 짓는 일*은 우리 시대의 새로운 건축입니다.
